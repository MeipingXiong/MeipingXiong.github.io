摘要

本报告基于 PIC（Paediatric Intensive Care）临床公开数据集，围绕 ICU 患者结局（二分类）预测任务开展数据分析与建模。首先对原始数据进行清洗去重，针对高缺失特征与信息过于稀疏的样本设置阈值筛除，并对数值与类别特征分别进行插补、标准化与 One-Hot 编码；同时采用按 SUBJECT_ID 分组的训练/测试集划分以避免数据泄漏。随后构建并比较逻辑回归、随机森林、梯度提升树及神经网络等多种模型，使用 Accuracy、Balanced Accuracy、Precision、Recall、F1、ROC-AUC 与 PR-AUC 等指标进行评估，并通过混淆矩阵、ROC 和 PR 曲线与校准曲线对模型行为进行可视化分析。实验结果表明，在类别不平衡的临床预测场景中需综合多指标与可视化诊断模型性能，才能更全面评估模型的实际效用与可靠性。

不同机器学习模型评价指标


我们在各个指标上评估了5种模型，从表 1 可以看出，不同模型在各评价指标上呈现出明显差异，这也反映了 ICU 结局预测任务中类别不平衡带来的典型现象：单看 Accuracy 容易产生误判，需要结合 Balanced Accuracy、Recall、F1 以及 PR-AUC 综合判断。首先，随机森林（RF）取得了最高的 Accuracy（0.9535），同时 ROC-AUC（0.8624）和 PR-AUC（0.2237）也较高，说明其在“排序能力”（把高风险样本排到更前面）方面表现强。但其 Recall 极低（0.0091）、F1 极低（0.0178），表明在默认阈值下几乎不预测正类，属于明显“保守预测”模型：虽然总体准确率很高，但对正类（高风险/死亡）样本的识别能力几乎不可用。因此，RF 的高 Accuracy 更可能来源于“多数类占比高”，而不是模型真正具备强的临床风险识别能力。相比之下，HGB在综合指标上更均衡：其 Accuracy（0.9286）仍然较高，同时 Precision（0.2814）与 Recall（0.3454）形成更合理的权衡，使得 F1 达到 0.3102（表中最高），并且 ROC-AUC（0.8397）与 PR-AUC（0.2161）也保持在较高水平。整体来看，HGB 更符合“不平衡二分类任务”对模型的实际需求：既具备较强区分能力，也能在一定程度上识别正类样本。在神经网络方法中，Table Neural Network 的 Balanced Accuracy（0.7049）最高，且 Recall（0.5454）也较高，说明它对正类更加敏感、更倾向于识别高风险样本；相应地，其 Precision（0.1639）不高，导致 F1（0.2521）略低于 HGB，但整体属于“更关注召回”的模型风格，适合需要尽量减少漏报的应用需求。相比之下，MLP 虽然 Accuracy（0.9518）较高，但 Balanced Accuracy（0.5380）与 F1（0.1363）较低，说明其在不平衡数据下仍然存在偏向多数类的倾向。

🚀 预测模型可视化分析

Logistic Regression（逻辑回归）


混淆矩阵：逻辑回归在负类样本上的预测较为稳定，大多数负类被正确分类；但在正类样本上仍存在较多漏报，正类召回率有限。这表明线性模型在当前特征空间下对高风险样本的区分能力不足，整体预测偏向保守。ROC 曲线：其 ROC 曲线整体位于对角线之上，但与树模型和神经网络相比提升有限，ROC-AUC 较低，说明逻辑回归在不同阈值下对正负样本的排序能力相对有限。Precision–Recall 曲线：在 PR 曲线中，随着召回率提高，精确率迅速下降，Average Precision 较低，反映出模型在正类稀缺的场景下难以同时保持较高的 Precision 与 Recall。Calibration 曲线：逻辑回归的校准曲线整体较为平滑，但多数区间位于理想对角线下方，表现出一定的概率低估现象。其概率输出在趋势上较为稳定，但并不能完全视为真实风险概率。

Random Forest（随机森林）


混淆矩阵：随机森林在默认阈值下几乎将所有样本预测为负类，负类预测准确率极高，而正类样本几乎全部被误判，正类召回率接近于零，表现出明显的保守预测倾向。 ROC 曲线：尽管在混淆矩阵上的分类结果较差，其 ROC 曲线却显著优于逻辑回归，ROC-AUC 较高，说明模型在连续评分层面具有较强的正负样本排序能力。 Precision–Recall 曲线：PR 曲线在低召回区间仍保持相对较高的精确率，Average Precision 较高，表明模型仅在极少数高置信度样本上预测为正，覆盖范围较窄。 Calibration 曲线：随机森林的校准曲线波动明显并偏离理想对角线，说明其原始概率输出未经过良好校准，不宜直接作为真实风险概率进行解释。

HistGradientBoosting（梯度提升树，HGB）


混淆矩阵：梯度提升树在负类预测上保持较高准确率的同时，能够识别出一定比例的正类样本，正类召回率明显优于随机森林，整体预测结果更加均衡。 ROC 曲线：其 ROC 曲线形态平滑且整体靠近左上角，ROC-AUC 较高，说明模型在不同阈值下均具备较强的区分正负样本能力。 Precision–Recall 曲线：PR 曲线在较宽的召回区间内维持相对稳定的精确率，Average Precision 较高，体现了该模型在不平衡数据场景下对正类样本的有效识别能力。 Calibration 曲线：校准曲线显示模型在中高概率区间仍存在一定偏差，概率输出尚未完全可靠，若用于风险分层仍需进一步校准处理。

MLP（多层感知机）


混淆矩阵：skMLP 在当前训练设置下预测结果高度偏向负类，正类样本几乎无法被识别，表现出明显的多数类偏置现象。 ROC 曲线：ROC 曲线整体优于随机猜测，但弱于梯度提升树与随机森林，说明模型在排序层面具有一定判别能力，但尚不足以支持有效分类决策。 Precision–Recall 曲线：PR 曲线整体较低，Average Precision 中等偏低，表明模型在正类稀缺条件下对高风险样本的有效识别区间较窄。 Calibration 曲线：校准曲线整体位于理想对角线下方，显示模型输出概率偏低，对正类风险存在系统性低估。

Tabular Neural Network（表格神经网络）


混淆矩阵：TorchMLP 在正类与负类之间取得相对平衡的预测结果，能够识别出较多正类样本，正类召回率明显高于随机森林与 skMLP，但仍存在一定比例的漏报。 ROC 曲线：其 ROC 曲线整体优于逻辑回归并接近树模型，ROC-AUC 表明该模型具备较强的整体判别能力。 Precision–Recall 曲线：PR 曲线显示模型在提高召回率的同时精确率下降相对平缓，Average Precision 处于较优水平，更适合强调正类识别能力的应用场景。 Calibration 曲线：校准曲线呈现连续但偏离理想对角线的趋势，说明模型输出概率可用于相对风险排序，但仍不宜直接解释为真实发生概率。
